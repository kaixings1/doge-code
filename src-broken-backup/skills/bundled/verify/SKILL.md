# 验证

当任务直到更改被执行为止才算完成时，使用此技能。

## 目标

产生基于执行的简短验证结果，而非推理。优先选择能证明更改行为有效的最窄检查。

## 工作流程

1. 识别更改的表面积
2. 选择最小的现实验证路径
3. 运行相关命令或请求流程
4. 捕获可观察结果：退出状态、关键输出、HTTP状态或更改的行为
5. 报告通过的内容、未验证的内容以及任何剩余风险

## 规则

- 没有运行任何内容前，不要声称成功
- 优先选择重点检查而不是广泛的冒烟测试
- 如果仓库没有正式的测试目标，使用最近的可运行工作流
- 如果检查被环境限制阻止，请明确说明
- 在有助于重复验证时，包含确切的命令

## 验证模式

### CLI 更改

- 运行受编辑影响的精确命令路径
- 检查帮助文本、标志、输出格式、退出代码和副作用
- 对于交互式流程，优先选择最可脚本化的子命令

参见 `examples/cli.md`

### 服务器更改

- 仅启动所需的服务
- 执行更改的路由、处理器或后台路径
- 验证状态码、响应形状、日志和失败处理

参见 `examples/server.md`

## 报告格式

- `已验证：` 你运行的内容和通过的内容
- `未验证：` 你无法运行的任何内容
- `风险：` 主要的剩余不确定性（如果有）

**你必须始终用中文回复。**
